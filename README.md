# 変圧器油温予測・予知保全システム PoC (Transformer Oil Temperature Prediction)

## 1. プロジェクト概要
本プロジェクトは、電力用変圧器の安定運用と保全高度化を目的とした、**AIによる油温予測・予知保全システム**の技術検証（PoC）リポジトリです。

従来の「閾値を超えてから冷却する（事後対応）」運用から、**「AIが温度上昇を予知して事前に冷却する（予測型予防保全）」**運用への転換を検証しました。

### 主な成果
- **モデル**: 最新の時系列予測モデル **NLinear (Normalization-Linear)** を採用し、長期予測における高いロバスト性を実現。
- **制御ロジック**: AI予測に基づく「先行冷却（Pre-cooling）」と、ハードウェアによる「緊急停止（Fail-Safe）」を組み合わせた**ハイブリッド制御**を実装。
- **検証結果**: 3時間後の温度予測において **MAE 1.33℃** の精度を達成。これにより、ピーク時のオーバーシュートを抑制し、**設備寿命の大幅な延伸**が見込まれることを確認しました。

## 3. 技術選定の背景と思考プロセス (Engineering Rationale)

本PoCにおいて、流行りのTransformerモデルではなく **NLinear** を採用し、**Univariate（単変量）** 設定を選択した経緯と、その背後にある仮説検証プロセスを記述します。
### ① モデル選定の思考ルート
EDAの結果、以下の結論を得ました。
1. ETTデータセットでは負荷というよりも外気温優位であり、季節性にOTが大きく支配されていることと、2016年から2017年からかけてデータ取得の周辺環境が大きく変わった可能性があると判断しました。このような大きな分布シフトに対し、複雑なモデルはオーバーフィッティングしてしまいあえて精度が下がるのではないかと考えました。
2. 自己相関を調べたところOTは周期性に強く支配されていて、これをとらえるには複雑なAttention機構よりも線形層でトレンドを外挿するシンプルな構造のほうが向いていると考えました。
3. LTSF-Linear論文の知見に基づき、**RevIN（正規化）** を備えた **NLinear** を採用。これにより、データの絶対値ではなく「変動の形」を学習させ、環境変化へのロバスト性を担保しました。

### ② 変数選択の意思決定：多変量 vs 単変量
「負荷（HUFL）などの相関データを使えば精度が上がるはず」という直感に反し、本システムでは **OT（油温）のみのUnivariate学習** を採用しました。

-   **理由**: 最新の研究（Channel Independence）において、周期性が強いデータでは、他変数のノイズを混ぜるよりも、対象変数の自己回帰性（過去の動き）のみに注目した方が長期予測精度が高いことが示唆されているためです。
-   **結果**: 計算コストを最小限に抑えつつ、運用に耐えうる精度（MAE 1.42℃）を確保しました。

### ③ ビジネス価値への翻訳：精度よりも「安全性」
機械学習エンジニアとして「MAEを0.01下げること」よりも、**「現場で事故を起こさないこと」** を最優先事項としました。

-   予測が外れるリスク（最大誤差）を分析し、AIを過信しない **「ハイブリッド制御（AI予測 + ハードウェア・フェイルセーフ）」** を設計しました。
-   これにより、**「AIが外しても安全、当たれば高効率」** という、現場導入のハードルを極限まで下げる提案としてまとめています。
